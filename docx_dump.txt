Resumo
O presente relatório descreve o desenvolvimento do BibliotecaTC, uma aplicação web full-stack de gestão de biblioteca, elaborada no âmbito da Prova de Aptidão Profissional do Curso Profissional de Programador de Informática da Escola Secundária Tomás Cabreira.
A aplicação oferece um portal de utilizador para explorar o catálogo e gerir empréstimos, e uma consola administrativa para gerir livros, categorias e utilizadores. Foi desenvolvida com React, Supabase e Tailwind CSS, com suporte a seis idiomas e temas claro/escuro.

Palavras-chave: Gestão de biblioteca; Aplicação web; React; Supabase; Full-stack; Autenticação

Abstract
This report describes the development of BibliotecaTC, a full-stack web library management system, produced as the final PAP project for the Professional Course in Computer Programming at Escola Secundária Tomás Cabreira.
The application provides a user portal for browsing the catalogue and managing loans, and an admin console for managing books, categories and users. Built with React, Supabase and Tailwind CSS, with support for six languages and light/dark themes.

Keywords: Library management; Web application; React; Supabase; Full-stack; Authentication

1. Introdução
O BibliotecaTC é um sistema de gestão de biblioteca desenvolvido como Prova de Aptidão Profissional. O projeto surgiu da necessidade de modernizar a gestão de livros e empréstimos de uma escola, substituindo processos manuais por uma solução digital eficiente e acessível.
Os objetivos principais são desenvolver uma plataforma web responsiva, implementar autenticação segura com controlo de acessos por papéis, e criar interfaces distintas para utilizadores e administradores. O relatório está organizado em enquadramento teórico, métodos, execução do projeto, conclusão e anexos.

2. Enquadramento Teórico
2.1. Contextualização do Projeto
A digitalização das bibliotecas escolares é uma tendência crescente, substituindo registos manuais por sistemas mais eficientes. O BibliotecaTC propõe uma solução web acessível em qualquer dispositivo, com separação clara entre ambiente de utilizador e consola administrativa.
2.2. Tecnologias e Ferramentas Utilizadas
React (v19) + Vite — framework de UI baseado em componentes com build rápido.
React Router v7 — roteamento para aplicações de página única (SPA).
Tailwind CSS (v4) — estilização utilitária e responsiva.
Framer Motion — animações e micro-interações na interface.
Supabase — BaaS com PostgreSQL, autenticação segura e dados em tempo real.
Vercel — Alojamento web.
Lucide React — ícones vetoriais acessíveis e consistentes.

3. Métodos e Técnicas Utilizadas
O desenvolvimento seguiu uma metodologia de trabalho-projeto com sprints semanais e ciclos iterativos de implementação e revisão.
3.1. Questão de Investigação
Como desenvolver um sistema de gestão de biblioteca web, seguro e completo, com tecnologias full-stack modernas, adequado a uma comunidade escolar?
3.2. Objetivos
Implementar uma aplicação web com interfaces distintas para utilizador e administrador.
Garantir autenticação segura e controlo de acessos por papéis (RBAC).
Desenvolver UI responsiva, acessível e internacionalizada.
Integrar descrições de livros geradas por inteligência artificial.
3.3. Procedimentos
Análise de requisitos e prototipagem das interfaces.
Desenvolvimento incremental a partir do núcleo do sistema.
Testes funcionais manuais com correção iterativa de erros.

4.1. Execução do Projeto
O projeto desenvolveu-se em fases: configuração do ambiente e base de dados; autenticação e RBAC; portal de utilizador; consola administrativa; e internacionalização e refinamento final da interface.

4.2. Funcionalidades Desenvolvidas
Catálogo pesquisável com filtros por categoria e descrições geradas por IA.
Gestão de empréstimos com prazos, histórico e notificações in-app.
Autenticação por email/palavra-passe, magic link e recuperação de conta.

5. Conclusão
O BibliotecaTC cumpriu os objetivos propostos, resultando numa aplicação robusta, escalável e com boa experiência de utilizador. A escolha de React e Supabase revelou-se acertada, permitindo um desenvolvimento ágil com funcionalidades avançadas.
Os pontos fortes incluem a arquitetura moderna, a internacionalização e a integração de IA. As principais dificuldades foram as políticas RLS no Supabase e a otimização mobile. Para trabalhos futuros sugere-se reservas de livros, notificações por email e exportação de relatórios em PDF.
